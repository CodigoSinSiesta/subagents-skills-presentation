# ¿Qué es un Skill?

## Tiempo estimado
3 minutos

## Puntos clave
- **Definición**: Paquete de instrucciones + contexto + herramientas para una tarea específica
- **No es un plugin**: Es conocimiento estructurado que el agente carga
- **Componentes de un skill**:
  - Instrucciones específicas (el "qué hacer")
  - Contexto relevante (documentación, ejemplos)
  - Herramientas disponibles (MCP servers, comandos)
  - Restricciones y reglas (el "qué NO hacer")
- **Analogía clave**:
  - Skill = Partitura musical
  - Subagente = Músico especializado
  - Orquestación = Director coordinando
- **Por qué skills, no prompts largos**:
  - Los prompts largos se ignoran ("lost in the middle")
  - Los skills se cargan ACTIVAMENTE antes de la tarea
  - Estructura vs. texto libre

## Datos relevantes
- Los skills pueden reducir el prompt de usuario un 80%
- Cargar el skill correcto mejora resultados un 50-70%
- Claude Code, OpenCode, Cursor todos usan sistemas de skills
